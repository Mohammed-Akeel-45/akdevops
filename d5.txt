Program 5: Install Jenkins on Local Environment and Configure Jenkins for First Use
Aim

To install Jenkins on a local machine using Docker and configure Jenkins for first use.

Software Requirements
Windows 10/11
Docker Desktop
Visual Studio Code
Internet Connection
Step 1: Create Project Folder

Create a folder:

C:\docker-projects\jenkins
Explanation

This folder stores all files required to create and run Jenkins using Docker.

Step 2: Open Folder in VS Code

Open VS Code.

Select:

File → Open Folder

Choose:

C:\docker-projects\jenkins
Explanation

VS Code makes it easier to create files without automatically adding .txt extensions.

Step 3: Create Dockerfile

Create a new file named:

Dockerfile

Paste:

FROM jenkins/jenkins:lts-jdk21

USER root

RUN apt-get update && \
    apt-get install vim -y && \
    apt-get install -y curl ca-certificates && \
    curl -fsSL https://deb.nodesource.com/setup_22.x | bash - && \
    apt-get install -y nodejs

USER jenkins
Explanation of Dockerfile
Base Image
FROM jenkins/jenkins:lts-jdk21

Uses the official Jenkins Long Term Support image with Java 21.

Switch to Root User
USER root

Provides administrative privileges to install software.

Update Package List
apt-get update

Updates package information from repositories.

Install Vim Editor
apt-get install vim -y

Installs Vim editor.

Install curl and SSL Certificates
apt-get install -y curl ca-certificates

Installs tools required to download packages securely.

Install Node.js
curl -fsSL https://deb.nodesource.com/setup_22.x | bash -

apt-get install -y nodejs

Installs Node.js version 22.

Return to Jenkins User
USER jenkins

Runs Jenkins using the default Jenkins user for security purposes.

Step 4: Create docker-compose.yml

Create a file named:

docker-compose.yml

Paste:

services:
  jenkins:
    build: .
    container_name: jenkins
    ports:
      - 8081:8080
      - 50000:50000
    networks:
      - jenkins-sonar-net
    volumes:
      - jenkins_home:/var/jenkins_home

  sonarqube:
    image: sonarqube:latest
    container_name: sonarqube
    ports:
      - 9000:9000
    networks:
      - jenkins-sonar-net
    volumes:
      - sonarqube_data:/opt/sonarqube/data
      - sonarqube_extensions:/opt/sonarqube/extensions
      - sonarqube_logs:/opt/sonarqube/logs

networks:
  jenkins-sonar-net:
    driver: bridge

volumes:
  jenkins_home:
    external: true
  sonarqube_data:
    external: true
  sonarqube_extensions:
    external: true
  sonarqube_logs:
    external: true
Explanation of docker-compose.yml
Jenkins Service
build: .

Builds Jenkins image using Dockerfile.

container_name: jenkins

Creates a container named "jenkins".

ports:
  - 8081:8080

Maps host port 8081 to Jenkins container port 8080.

Jenkins can be accessed at:

http://localhost:8081
volumes:
  - jenkins_home:/var/jenkins_home

Stores Jenkins configuration permanently.

SonarQube Service
image: sonarqube:latest

Downloads the latest SonarQube image.

ports:
  - 9000:9000

Allows access to SonarQube through:

http://localhost:9000
Network
networks:
  jenkins-sonar-net:
    driver: bridge

Creates a bridge network to enable communication between Jenkins and SonarQube.

Volumes
jenkins_home

Stores Jenkins jobs and plugins.

sonarqube_data

Stores SonarQube database and analysis reports.

sonarqube_extensions

Stores SonarQube plugins.

sonarqube_logs

Stores SonarQube log files.

Step 5: Create Volumes

Open Command Prompt and execute:

docker volume create jenkins_home

docker volume create sonarqube_data

docker volume create sonarqube_extensions

docker volume create sonarqube_logs
Explanation

Volumes provide persistent storage so that data remains available even after containers are removed.

Step 6: Navigate to Project Folder
cd C:\docker-projects\jenkins
Explanation

Changes the working directory to the folder containing Docker files.

Step 7: Start Containers

Execute:

docker compose up -d
Explanation

Docker Compose:

Builds Jenkins image.
Downloads SonarQube image.
Creates network.
Starts Jenkins container.
Starts SonarQube container.
Step 8: Verify Containers

Execute:

docker ps
Explanation

Displays all running containers.

Step 9: Obtain Jenkins Password

Execute:

docker exec jenkins cat /var/jenkins_home/secrets/initialAdminPassword
Explanation

Retrieves the initial administrator password generated by Jenkins.

Step 10: Open Jenkins

Open a browser and enter:

http://localhost:8081
Step 11: Unlock Jenkins

Paste the copied password and click:

Continue
Step 12: Install Suggested Plugins

Select:

Install Suggested Plugins
Explanation

Jenkins installs commonly used plugins such as:

Git Plugin
Pipeline Plugin
Maven Plugin
Credentials Plugin
SSH Plugin
Step 13: Create Administrator Account

Provide:

Username
Password
Full Name
Email Address

Click:

Save and Continue
Step 14: Configure Jenkins URL

Keep:

http://localhost:8081

Click:

Save and Finish
Step 15: Start Using Jenkins

Click:

Start using Jenkins

The Jenkins Dashboard appears.

Result

Jenkins was successfully installed and configured for first use using Docker and Docker Compose.
