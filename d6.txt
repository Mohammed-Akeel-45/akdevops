Program 6: Install and Configure SonarQube and Create a Jenkins CI/CD Pipeline for Static Code Analysis
Aim

To install and configure SonarQube on a local machine and create a Jenkins CI/CD pipeline that integrates SonarQube to perform static code analysis.

Software Requirements
Windows 10/11
Docker Desktop
Visual Studio Code
Internet Connection
Theory
Jenkins

Jenkins is an open-source automation server used for Continuous Integration and Continuous Delivery (CI/CD). It automates build, testing, and deployment processes.

SonarQube

SonarQube is an open-source platform used to inspect source code and detect:

Bugs
Vulnerabilities
Code Smells
Duplicate Code
CI/CD Pipeline

A pipeline is a sequence of stages that automate software development activities.

Architecture

Developer Code

↓

Jenkins Pipeline

↓

Build Application

↓

SonarQube Analysis

↓

Code Quality Report

Step 1: Create Project Folder

Create:

C:\docker-projects\jenkins-sonarqube
Explanation

This folder stores Docker files required to run Jenkins and SonarQube.

Step 2: Open Folder in VS Code

Open VS Code.

Select:

File → Open Folder

Choose:

C:\docker-projects\jenkins-sonarqube
Step 3: Create Dockerfile

Create a file named:

Dockerfile

Paste:

FROM jenkins/jenkins:lts-jdk21

USER root

RUN apt-get update && \
    apt-get install vim -y && \
    apt-get install -y curl ca-certificates && \
    curl -fsSL https://deb.nodesource.com/setup_22.x | bash - && \
    apt-get install -y nodejs

USER jenkins
Explanation of Dockerfile
FROM jenkins/jenkins

Uses Jenkins Long Term Support image with Java 21.

USER root

Switches to administrator privileges.

apt-get update

Updates package information.

apt-get install vim -y

Installs Vim editor.

curl -fsSL https://deb.nodesource.com/setup_22.x | bash -

Adds Node.js repository.

apt-get install -y nodejs

Installs Node.js version 22.

USER jenkins

Returns to Jenkins user.

Step 4: Create docker-compose.yml

Create:

docker-compose.yml

Paste:

services:
  jenkins:
    build: .
    container_name: jenkins
    ports:
      - 8081:8080
      - 50000:50000
    networks:
      - jenkins-sonar-net
    volumes:
      - jenkins_home:/var/jenkins_home

  sonarqube:
    image: sonarqube:latest
    container_name: sonarqube
    ports:
      - 9000:9000
    networks:
      - jenkins-sonar-net
    volumes:
      - sonarqube_data:/opt/sonarqube/data
      - sonarqube_extensions:/opt/sonarqube/extensions
      - sonarqube_logs:/opt/sonarqube/logs

networks:
  jenkins-sonar-net:
    driver: bridge

volumes:
  jenkins_home:
    external: true
  sonarqube_data:
    external: true
  sonarqube_extensions:
    external: true
  sonarqube_logs:
    external: true
Explanation
Jenkins Service

Creates Jenkins container.

SonarQube Service

Creates SonarQube container.

Ports

8081 → Jenkins

9000 → SonarQube

Network

Allows communication between Jenkins and SonarQube.

Volumes

Provide persistent storage for Jenkins and SonarQube data.

Step 5: Create Volumes

Open Command Prompt and execute:

docker volume create jenkins_home

docker volume create sonarqube_data

docker volume create sonarqube_extensions

docker volume create sonarqube_logs
Explanation

Volumes preserve data even after containers are deleted.

Step 6: Start Containers

Navigate to the folder:

cd C:\docker-projects\jenkins-sonarqube

Run:

docker compose up -d
Explanation

Docker Compose:

Builds Jenkins image.
Downloads SonarQube image.
Creates network.
Starts both containers.
Step 7: Verify Running Containers

Execute:

docker ps

Two containers should be visible:

jenkins
sonarqube
Step 8: Obtain Jenkins Password

Execute:

docker exec jenkins cat /var/jenkins_home/secrets/initialAdminPassword

Copy the generated password.

Step 9: Open Jenkins

Open:

http://localhost:8081

Paste the password.

Step 10: Install Suggested Plugins

Choose:

Install Suggested Plugins

Step 11: Create Admin User

Provide:

Username
Password
Full Name
Email Address

Click:

Save and Continue

Step 12: Open SonarQube

Open:

http://localhost:9000

Default credentials:

Username:

admin

Password:

admin

Change the password after login.

Step 13: Generate SonarQube Token

Navigate:

My Account

↓

Security

↓

Generate Tokens

Name:

jenkins-token

Click:

Generate

Copy the token.

Step 14: Install SonarQube Scanner Plugin in Jenkins

Navigate:

Manage Jenkins

↓

Plugins

↓

Available Plugins

Search:

SonarQube Scanner

Install the plugin.

Restart Jenkins.

Step 15: Configure SonarQube Server

Navigate:

Manage Jenkins

↓

System

↓

SonarQube Servers

Click:

Add SonarQube

Name:

My SonarQube Server

Server URL:

http://sonarqube:9000

Authentication Token:

Paste the generated token.

Save.

Step 16: Configure Maven

Navigate:

Manage Jenkins

↓

Tools

↓

Maven Installations

Add Maven

Name:

Maven3

Check:

Install Automatically

Save.

Step 17: Create Pipeline Job

Dashboard

↓

New Item

↓

Pipeline

Name:

SonarPipeline

Click OK.

Step 18: Pipeline Script

Select:

Pipeline Script

Paste:

pipeline {
    agent any

    tools {
        maven 'Maven3'
    }

    stages {

        stage('Build') {
            steps {
                sh 'mvn clean install'
            }
        }

        stage('SonarQube Analysis') {
            steps {
                withSonarQubeEnv('My SonarQube Server') {
                    sh 'mvn sonar:sonar'
                }
            }
        }

    }
}

Save.

Step 19: Build the Pipeline

Click:

Build Now

Step 20: View Analysis Report

Open:

http://localhost:9000

View:

Bugs
Vulnerabilities
Code Smells
Duplicated Code
Result

SonarQube was successfully installed and integrated with Jenkins. A CI/CD pipeline was created to perform static code analysis and generate code quality reports.
